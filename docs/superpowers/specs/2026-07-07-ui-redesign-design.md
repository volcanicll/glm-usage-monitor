# GLM Usage Panel UI 重构设计

> 日期：2026-07-07
> 范围：`src/views/UsagePanel.ts`（Webview HTML/CSS/JS）
> 目标：在不动数据层的前提下，统一视觉系统、优化布局节奏、提升数字与状态可读性。

## 背景与问题

当前面板（v0.0.7）功能完整，包含：配额横条、模型占比环形图、工具使用柱状图、Token 趋势折线图。但存在三类问题：

1. **视觉层级弱**：颜色散落、卡片缺乏层次感、字号梯度不明显，关键数字不够突出。
2. **布局拥挤**：三块内容平铺，主次不分；窄宽下空间利用不佳。
3. **可读性不足**：数字格式不统一、时间缺乏相对感知、状态警示弱。

## 目标与非目标

**目标**
- 建立统一设计令牌（颜色 / 圆角 / 间距 / 字号），让整体观感更现代清爽。
- 优化布局节奏，强化主次，适配窄宽。
- 提升数字可读性与状态感知（进度色、相对时间、警示态）。

**非目标**
- 不改变信息架构（仍是 配额 → 主网格 → 趋势 → 页脚）。
- 不改动数据层（`QuotaSummary`、`GLMUsageService` 等）。
- 不引入第三方图表库（继续用内联 SVG，保持零依赖）。

## 视觉风格基调

**现代清爽（Stripe / Linear / Vercel Analytics 风格）**：克制的留白、柔和圆角、细边框、数据为主的中间调，与 VSCode 主题良好融合。

## 设计方案

### ① 视觉系统：统一设计令牌

替换当前散落的硬编码值，建立语义化 CSS 变量：

```css
/* 层级背景 */
--surface-1: var(--vscode-editor-background);          /* 页面底 */
--surface-2: var(--vscode-editor-inactiveSelectionBackground); /* 卡片 */
--surface-3: var(--vscode-editor-selectionBackground); /* 卡片内嵌区块 */

/* 主色梯度（基于 #6366f1） */
--accent: #6366f1;
--accent-soft: rgba(99,102,241,.12);

/* 语义色 */
--ok: #10b981;
--warn: #f59e0b;        /* 黄 */
--warn-strong: #d9a441;
--danger: #ef4444;
--danger-soft: #d05d5d;

/* 圆角 / 间距 */
--radius-sm: 6px; --radius-md: 10px; --radius-lg: 14px;
--space-1: 4px; --space-2: 8px; --space-3: 12px;
--space-4: 16px; --space-5: 20px; --space-6: 24px;

/* 字号梯度 */
--text-xs: 10px; --text-sm: 11px; --text-base: 12px;
--text-md: 13px; --text-lg: 16px; --text-xl: 22px; --text-2xl: 28px;
```

**卡片质感**：1px 半透明边框 + surface-2 背景 + `--radius-lg` 圆角 + 内边距 16→20px。

**图标**：每个卡片标题前加内联 SVG 图标（环形图、工具、趋势），统一 14×14、`currentColor`。

### ② 布局：信息分层与空间重分配

整体保持单列流式，强化主次：

```
┌─────────────────────────────────────┐
│ Header（套餐 + tabs + 刷新）          │  ← 加套餐图标
├─────────────────────────────────────┤
│ 配额横条（保留，优化字号/颜色/警示态） │
├──────────────────┬──────────────────┤
│ 模型占比环形图     │  工具使用统计      │
├──────────────────┴──────────────────┤
│ Token 用量趋势折线图（全宽，层级提升）  │
├─────────────────────────────────────┤
│ Footer                               │
└─────────────────────────────────────┘
```

**关键调整**

- **配额横条**：保留横向布局，但
  - 百分比字号 13→16px 加粗；
  - 接近上限（≥80%）时整条加黄色左边框 + 浅黄背景警示；
  - 离线 badge 样式优化（更柔和的填充）。

- **模型占比环形图**：
  - 直径 180→200px，中心数字字号 22→28px；
  - 图例从纯文字升级为带「迷你占比条」的横条：每行 = 色点 + 模型名 + 用量 + 占比 + 一条 100% 宽的细进度条（按占比填色）。比当前 4 列 grid 信息密度更高。

- **工具使用柱状图**：
  - 柱体加圆角，右侧除计数外加占比百分比；
  - 计数卡片弱化（保留但缩小），避免与柱状图信息重复；
  - 空数据态加占位图形 + 友好文案。

- **趋势折线图**（重点提升）：
  - 加水平虚线网格；
  - Y 轴顶部加「Token」单位标注；
  - 数据点加小圆点标记；
  - 悬停显示 tooltip（日期 + 模型 + 数值，用原生 `<title>` 或轻量 JS）；
  - 图例项可点击切换显隐（轻量 JS，可选）。

- **响应式**：新增 760px 断点
  - `> 760px`：主网格 2 列；
  - `480–760px`：主网格 1 列，配额横条横排；
  - `< 480px`：配额横条堆叠为多行。

### ③ 可读性：数字与状态优化

- **数字格式统一**
  - Token：沿用 `formatTokenCount`（1.2K / 3.4M / 1.1B），保持现状。
  - 百分比：整数（如 35%），趋势图例外可用一位小数。
  - 剩余量：加语义色——绿（<60%）、黄（60–80%）、红（>80% 已用），复用 `getProgressColor`。

- **时间格式**
  - 重置时间：保持绝对时间显示，悬停 tooltip 加相对时间（"3 小时后重置"）。
  - 刷新信息：沿用相对时间（"刚刚 / N 分钟前"）。

- **状态指示**
  - 配额 ≥80%：配额条黄色警示态。
  - 配额 ≥95%：红色危急态。
  - 离线模式：badge 文案 + 样式优化。

- **空状态**
  - 每个图表区块统一空态：占位 SVG 图形 + 友好文案，而非纯灰文字。

## 不变项（明确）

- 不改 `QuotaSummary` 数据结构与 `GLMUsageService`。
- 不改时间范围 tabs（today / last7Days / last30Days）。
- 不改 webview 消息协议（`refresh` / `changeRange`）。
- 不引入任何 npm 依赖。

## 实现位置

全部改动集中在 [src/views/UsagePanel.ts](src/views/UsagePanel.ts)：
- `<style>` 段：替换/扩充 CSS 变量与样式类。
- `getHtml()`：调整 HTML 模板（图例结构、柱状图、配额条警示态、图标）。
- `generateLineChartSection()`：加网格线、单位标注、数据点、tooltip。
- `generateDonutSvg()`：直径与中心字号调整。
- 新增私有方法：图标生成（`getIcon(name)`）、相对时间（`getRelativeTime(dateStr)`）、图例占比条 HTML。

## 验证

- 在 VSCode 中打开面板，分别切换 light / dark 主题，确认对比度与可读性。
- 切换 today / last7Days / last30Days，确认折线图与图表正常。
- 拉窄 webview 至 480px，确认响应式堆叠。
- 构造空数据场景（断网 / 新账号），确认空态展示。
